# Contributing

Thank you for your interest in contributing to this project.

This repository is a personal educational project focused on data structures, algorithms, and related exercises. Contributions that improve clarity, correctness, maintainability, and learning value are welcome.

## How You Can Contribute

You can contribute by:

- reporting bugs or unexpected behavior
- suggesting improvements to existing implementations
- proposing new algorithm or data structure exercises
- improving documentation
- fixing test cases or adding new ones
- refining code style, readability, or performance

## Before You Contribute

Please keep the following in mind:

- Be respectful and constructive in all communication
- Follow the project's academic integrity principles
- Do not submit copied coursework or material that violates academic rules
- Keep changes focused and easy to review

## Suggested Workflow

1. Fork the repository
2. Create a new branch for your change
3. Make your edits
4. Run the tests
5. Open a pull request with a clear description

## Code Style

Please try to keep contributions consistent with the existing codebase:

- use clear and descriptive names
- keep methods focused and readable
- prefer simple, well-tested solutions
- add or update tests when behavior changes

## Testing

If your contribution affects implementation logic, please make sure the relevant tests pass.

If possible, add new tests for:

- edge cases
- boundary conditions
- unexpected inputs
- new functionality

## Pull Request Guidelines

A good pull request should include:

- a short summary of the change
- why the change was needed
- any relevant test updates
- notes about assumptions or special cases

## Reporting Issues

If you find a problem, please include:

- what you expected to happen
- what actually happened
- steps to reproduce the issue
- any relevant inputs, outputs, or screenshots

## Respectful Collaboration

Please help keep this project welcoming and friendly. Questions, suggestions, and corrections are all appreciated when shared respectfully.

## Thank You

Your time and effort are appreciated. Contributions help make the project more useful for learning and reference.
